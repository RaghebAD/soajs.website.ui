Generic Content Service

SOAJS Generic Content Service is a unique service whose configuration & schema is stored and loaded from the core database. Using the Dashboard UI, navigate to Content Builder and create/manage the GCS entries.

GCS is used to deploy multiple services using the same code, simply create an entry in Content Builder, grab the name and version, provide them as environment variables and start the service. Repeat the same procedure for every GCS service you want to deploy. 

# deploying 2 different GCS services
$ npm install soajs.GCS

# set the GCS name and version
$ export SOAJS_GC_NAME=myservice1
$ export SOAJS_GC_VERSION=2
$ cd soajs.GCS
$ node .

# set another GCS name and version
$ export SOAJS_GC_NAME=myservice2
$ export SOAJS_GC_VERSION=5
$ cd soajs.GCS
$ node .

Features

GCS is completely manageable from the Dashboard UI, via a simple to use and rich UI under Bontent Buidler module. The module provides the ability to:

  • Create multiple GCS schemas with version support
  • Configure a GCS to work on different environments
  • Add & Manage Inputs of different types
  • Add & Manage APIs of different types and workflow

Versioning

Once a Schema is generated, the dashboard service takes care of the versioning and stores the schema in the database. Every Schema has a version, and this is crucial since an update at a later stage might affect the data of that service. Versions are bumped based on major updates:

  1. Global Service And Database configuration modification.
  2. Existing Inputs Configuration modification.
  3. Removing Existing Inputs.
  4. Existing APIs Configuration or workflow modification.
  5. Removing Existing APIs.

Deployment

If you deploy SOAJS using Docker, once a GCS has been created and stored in the database, simply head to host deployment under the Dashboard Environment and deploy a new Host for this service.

If you have deployed SOAJS manually, then you need to execute the following commands for every GCS service you want to deploy: 

# set another GCS name and version
$ export SOAJS_GC_NAME=myservice
$ export SOAJS_GC_VERSION=1
$ cd soajs.GCS
$ node .

Data Management

GCS Data is managed using the Dashboard UI under Content Management module. Once a GCS service is deployed, the module picks up its presence and displays an entry on its page. Head to Content Management page and click on the name of the GCS service. The module will load the schema of the GCS as you configured it in Content Builder, it will render it and display the UI interface with the allowed functionality accordingly.

You will also notice that if you configure a GCS to have its data stored in multiple Environments, then Content Management will display the data of that service based on these configured environments so you know the location of the data you are modifying.
