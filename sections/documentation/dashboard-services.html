Dashboard SOAJS


The Dashboard is a service API that allows you to manage and configure SOAJS services and components.

  1. Manage all your products in Productization section.
  2. Manage all your tenants and keys in Multitenancy section.
  3. Define Multiple Environments.
  4. Create applications, keys and external service API keys.

Dashboard Services

The dashboard service covers 3 sections needed for you application to start.
The dashboard gives you access to Environments, Productization and Multitenancy.


Environments

Create and manage environments as you desire.
Simply hit the api of SOAJS Dashboard and provide the inputs needed to create a new environment.
You can also perform similar requests to update an existing environment or remove it.

Once your environments are added and ready to be used, use the Agent to push them to the Registry.


Productization

Create and manage the products and their packages.
Products contain packages. These package are used by tenants' applications to function.
Packages provide Access Level control over the API services.

Checkout the Productization section in Provisioning to learn more about the schemas and functionality of products and packages.


Multitenancy

Create and manage the tenants and their applications.
Add oAuth security to your tenants if you need to.

Tenants applications contains keys. These keys provide the services you created with data that should be loaded from the database.
This data is specific per tenant such as templates location and headlines, mail configuration, URAC links ...etc.

Keys offer the ability to generate external keys that are used in the header of requests made to the services.
These keys have expiry dates and device and geo location security.

oAuth security is a simple configuration that later on should be used with the oAuth service of SOAJS.
The service will compare the credentials passed at login with the values of the tenant to determine if access should be granted or not.

Checkout the Multitenancy section in Provisioning to learn more about the schemas and functionality of tenants.

Usage

SOAJS Dashboard is located in soajs.dashboard.
Install:

$ npm install -d soajs.dashboard

Then navigate to the directory where it is installed and run the service.
$ cd soajs.dashboard
$ node index

The service is now running and listening http://127.0.0.1:4000/dashboard/.
