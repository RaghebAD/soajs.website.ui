Dashboard

The Dashboard is a service that allows you to manage and configure SOAJS services, hosts, core database information in all running environments. The dashboard is provided in two parts: service - UI.

The service provides the APIs needed to update the core database information in all environments. The UI is an interface built with AngularJS that gives you a rich interface to call the service APIs.

Installation

SOAJS Dashboard is located in soajs.dashboard. 

$ cd /opt/soasj/node_modules/
$ npm install soajs.dashboard
$ cd soajs.dashboard
$ node .

Set It Up

To set up the dashboard, you need to deploy either the development environment as explained under Deployment section under documentation or deploy the dashboard environment from Cloud Awareness.

If you want to set up the dashboard manually, you can find the needed steps in development environment under documentation section. The development environment aims to install all the needed services and start the dashboard with its UI on one machine which is ideal for development. To set up the dashboard, few steps are needed:

  1. Import the data from soajs.utilities to obtain a default administrator account, a tenant and a group that grant you access to the dashboard service so you can modify the data in core.
  2. Install and run the following services: Controller - Urac - Dashboard
  3. Install Nginx and create a virtual host pointing out the location of the dashboard ui folder as explained in details under Manual Installation in development environment.
  4. Login to the dashboard with account admin:
    Username: admin
    Password: password

Dashboard Services

The Dashboard service provides the following features:

  1. Manage your Products & Packages in Productization section.
  2. Manage your Tenants, Applications and keys in Multitenancy section.
  3. Manage the Environments, the databases & the clusters.
  4. Monitor the hosts on the cloud and execute maintenance operations.
  5. Monitor registered and running services.
  6. Manage Dashboard Users accounts and their groups.

Dasboard UI

The following snapshots are taken from Dashboard UI to provide you with an idea on the features you can benefit from if you install it and use it.

Profile Management

Logged in Users can update their account settings and profile information.



Users & Groups Management

Administrators have the ability to manage users and groups accounts that have access to login to the dashboard.



Multi Environments Management

TechOps & DevOps benefit from multi environment to monitor and manage databases, hosts and all environment properties.



Environments Clusters Management

TechOps & DevOps also benefit from multi environment to monitor and control services hosts.



Productization

Create and manage products, packages that apply access to services and are used by tenants.



Product Package Access Levels Management

Configure and customize the package access levels and permissions on your services.



Multitenancy

Create and manage tenants, applications, keys, and custom service environment configuration as well as override default package access levels where needed all from one place.



Tenant oAuth Users Management

Create and manage oauth users accounts based on your tenants.
