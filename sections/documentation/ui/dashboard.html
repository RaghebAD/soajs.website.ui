Dashboard

The Dashboard is a service that allows you to manage and configure SOAJS services, hosts, core database information in all running environments. The dashboard is provided in two parts: service - UI.

The service provides the APIs needed to update the core database information in all environments. The UI is an interface built with AngularJS that gives you a rich interface to call the service APIs.

Installation

SOAJS Dashboard is located in soajs.dashboard.
You can install the Dashboard by following the instruction in the deployment section either Manually or via Docker.


Dashboard Services

The Dashboard service provides the following features:

  1. Manage your Products & Packages in Productization section.
  2. Manage your Tenants, Applications and keys in Multitenancy section.
  3. Manage the Environments, the databases & the clusters.
  4. Monitor the hosts on the cloud and execute maintenance operations.
  5. Monitor registered and running services.
  6. Manage Dashboard Users accounts and their groups.

Dasboard UI

The Dashboard UI is built using AngularJS and provides a rich Graphical Interface that you can use to call the APIs of the Dashboard Service. Here are some of the features provided in the UI:

Guest Section

The UI Guest section offers the following:

  • Login:  Members can login to the dashboard.
  • Forgot Password:  Members can ask to change their password if forgotten.
  • Register:  Users can register to become members and receive email notification to validate the registration.

Profile Management

Once logged in, under My Account, members can update their account and profile information:

  • Edit Profile: Members can update their account and/or Profile information.
  • Change Password:  Members can change their password.
  • Change Email:  Members can change their email address.


Users & Groups Management

Administrators have the ability to manage users and groups accounts:

  • Accounts:  Administrators can create and manage users accounts.
  • Groups:  Administrators can create and manage users groups.
  • Link to Groups:  Administrators can add users to group(s).


Multi Environments Management

TechOps & DevOps benefit from multi environment to monitor and manage databases, hosts and all environment properties:

  • Environment:  Create and Manage multiple environments.
  • Databases:  Create and Manage the databases of your application.
  • Clusters:  Create and Manage the clusters used by the databases of your application.
  • Hosts:  Monitor and perfrom heartbeat, reloadRegistry and other maintenance operations on your services.


Productization

Create and manage products, packages that apply access to services and are used by tenants:

  • Products:  Create and manage products and add packages to them.
  • Packages:  Create and manage packages and add specify the ACL over your services.


Product Package Access Levels Management

Configure and customize the package access levels and permissions on your services.



Multitenancy

Create and manage tenants, applications, keys, and custom service environment configuration as well as override default package access levels where needed all from one place:

  • Tenants:  Create and manage tenants and specify their oAuth properties if needed.
  • oAuth Users:  Create and manage the tenants oAuth users accounts.
  • Applications:  Create and manage applications inside tenants. Assign the application product packages or choose to customize their ACL information.
  • Keys:  Create and manage application keys, external keys and their security and tenant services configuration.


Tenant oAuth Users Management

Create and manage oauth users accounts based on your tenants.
