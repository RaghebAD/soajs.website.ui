Installation

SOAJS requires that your machine has the following technologies installed and running so that you can download it and start using it:
- MongoDB ( version 2.6.7 or higher)
- NodeJS (version 0.10.25 or higher)

First you need to install all the needed modules of SOAJS and their dependencies as mentioned in Get Started Section under How to Install SOAJS.

  • soajs.controller 
    $ npm install soajs.controller
  • soajs.agent 
    $ npm install soajs.agent
  • soajs.oauth 
    $ npm install soajs.oauth
  • soajs.urac 
    $ npm install soajs.urac
  • soajs.examples 
    $ npm install soajs.examples
  • soajs.dashboard 
    $ npm install soajs.dashboard
Once all the modules are installed, run their services:
$ cd soajs.controller
$ node .
$ cd ../soajs.agent
$ node .
$ cd ../soajs.oauth
$ node .
$ cd ../soajs.urac
$ node .
$ cd ../soajs.dashboard
$ node .
Make sure that you have nginx installed and configured as mentioned in soajs.utilities/nginx.
Basically nginx configuration for SOAJS creates 2 virtual hosts the first being and address for the controller referred to as 127.0.0.1 and the second being and address to open the dashboard referred to as dashboard.soajs.org.
#virtual host for controller
server {
	server_name  api.soajs.org;
	location / {
		proxy_pass          http://soajs.controller;
		proxy_set_header    X-Forwarded-Proto 	    $scheme;
		proxy_set_header    Host             	    $http_host;
		proxy_set_header    X-NginX-Proxy     	    true;
		proxy_set_header    Connection        	    "";
	}
}
# virtual host for dashboard UI
server {
	server_name  dashboard.soajs.org;
	location / {
		root   /opt/soajs/node_modules/soajs.dashboard/ui/;
		index  index.html index.htm;
	}
}

Import Test Data

Now that all the needed modules are installed, nginx is configured, import the sample data to the database. Navigate to soajs.example/tools in your Terminal, and you will find a mongo script that imports some test data for you to start with.

# make sure soajs.mongo.sh is executable
$ chmod +x soajs.mongo.sh

# execute soajs.mongo.sh
$ ./soajs.mongo.sh

The script will import some test data so that you can login to the dashboard and start using it.
Note: You can always use the dashboard without importing the test data.


Using the Dashboard UI

Since nginx has been configured, you can now open your browser and use the Dashboard UI by going to: http://dashboard.soajs.org.

Login using the test administrator account, this account has full privilege to all features, and create the environments, products, tenants and users you want.
The dashboard UI is configured to work with the dashboard service by providing the test tenant key that has access to all services. You can modify the value of this key after you create some real tenants. Simple navigate to soajs.dashboard/ui/config.js and change the value of the key in that file.

Dashboard UI Snapshots

The Dashboard interface is built on AngularJS.
Benefit from the easy of use that the dashboard UI offers to easily navigate and manage SOAJS topology and provisioned data.

Menus and Grids enable you navigate, browse, search and locate, sort and find the different sections and records to accomodate the change that is needed whether to modify environments, products, tenants, users and other sections.

Quick links to Add, Edit and Remove content and/or configuration provide GUI interfaces and forms that once submitted will interact with the dashboard service, URAC and other services APIs.

Interactive notification messages keep you alerted of the status of every request made whether successful or failure and in case of failure, they provide a clarifying message of the error that occured.
